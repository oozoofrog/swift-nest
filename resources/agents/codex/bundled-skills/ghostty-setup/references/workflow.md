# Ghostty 설정 작업 절차

## 목적

Ghostty 설정 변경을 요청받았을 때, 오류와 회귀를 줄이면서 결과를 검증 가능한 형태로 전달하기.

## 1. 위치와 구조 확인하기

1. 설정 루트를 식별하기.
2. 엔트리 파일(`config`)에서 `config-file` 선언을 모두 찾기.
3. profile과 theme로 이어지는 include 체인을 순서대로 기록하기.

예시 명령:

```bash
pwd
rg -n '^config-file\\s*=' config profiles/*.conf themes/*.conf
```

## 2. 변경 목표를 옵션으로 분해하기

1. 사용자 요청을 기능 단위로 쪼개기.
2. 각 기능을 실제 Ghostty 키로 매핑하기.
3. 이번 변경에서 건드릴 파일을 최소화하기.

예시:

- "글자가 작다" -> `font-size`
- "대비가 약하다" -> `minimum-contrast`, `foreground/background`, `selection-*`
- "커서가 산만하다" -> `cursor-style`, `cursor-style-blink`

## 3. 수정 전에 기준선 확보하기

1. 수정 대상 파일의 현재 값을 읽어 핵심 라인을 메모하기.
2. 동일 키가 여러 파일에서 중복되는지 확인하기.
3. include 순서 기준으로 최종 우선순위를 판단하기.

## 4. 변경 적용하기

1. 공통값은 base profile, 변동값은 active profile/theme에 두기.
2. 의미 없는 정렬 변경은 피하고 필요한 라인만 수정하기.
3. 한 번에 한 목적만 반영해 원인 추적 가능성을 유지하기.

## 5. 검증하기

1. 경로 검증: `config-file`가 실제 파일을 가리키는지 확인하기.
2. 중복 검증: 같은 키가 충돌하는지 점검하기.
3. 체감 검증: 사용자가 즉시 확인할 항목을 제시하기.

체감 검증 체크리스트 예시:

- 터미널 재로드 후 글자 크기 적용 여부
- 배경/전경 대비와 선택 영역 가독성
- 커서 모양 및 깜빡임 동작
- ANSI 색상(빨강/파랑/노랑) 과채도 여부

## 6. 보고 형식 유지하기

응답에 다음 항목을 포함하기:

1. 변경 파일 경로 목록
2. 핵심 옵션 변경 요약(이전 -> 이후 또는 의도 설명)
3. 사용자 검증 단계
4. 실패 시 롤백 방법(어떤 파일/라인을 되돌릴지)

## 7. 문제 유형별 우선 점검

- 값이 적용되지 않음:
  - include 순서
  - 동일 키 중복 선언
  - 잘못된 상대 경로
- 눈 피로:
  - `font-size`
  - `minimum-contrast`
  - 선택 영역 대비
  - 커서 깜빡임
- 테마만 바꾸고 싶음:
  - profile에서 `config-file = "../themes/..."` 라인만 우선 조정하기
